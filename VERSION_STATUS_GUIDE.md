# 版本状态功能说明

## 🎯 新增功能

为每个JSON模板文件新增了**版本状态**属性，支持标记模板为"新"或"旧"版本，并提供筛选功能。

## 📋 版本状态属性

### 状态说明
- **New（新）**: 当前正在使用的新版本模板
- **Old（旧）**: 旧版本模板，可能已废弃或不再推荐使用

### 默认值
- 新增的模板文件默认版本状态为 **New**
- 已有模板文件升级到新版本后，默认为 **New**

## 🔍 筛选功能

### 1. 搜索页面筛选
访问 http://localhost:5000

**使用方法**:
1. 在搜索表单中找到 **Version** 下拉框
2. 选择筛选条件：
   - **All**: 显示所有版本
   - **New**: 只显示新版本模板
   - **Old**: 只显示旧版本模板
3. 输入字段名称并搜索
4. 结果表格中会显示每个模板的版本状态（绿色=New，橙色=Old）

### 2. 模板列表页面筛选
访问 http://localhost:5000/templates

**使用方法**:
1. 在页面顶部找到 **Filter by Version** 下拉框
2. 选择筛选条件：
   - **All**: 显示所有模板
   - **New**: 只显示新版本模板
   - **Old**: 只显示旧版本模板
3. 页面自动刷新显示筛选结果
4. 表格中 **Version** 列显示版本状态徽章

## ✏️ 编辑版本状态

### 单个编辑
1. 在搜索结果或模板列表中找到目标模板
2. 点击 **Edit** 按钮
3. 在弹出框中找到 **Version Status** 字段
4. 选择 **New** 或 **Old**
5. 点击 **Save** 保存

### 批量编辑
1. 访问模板列表页面
2. 勾选需要修改的模板
3. 点击 **Batch Edit Selected (n)** 按钮
4. 在弹出框中找到 **Version Status (optional)** 字段
5. 选择：
   - **Keep original**: 保持原版本状态不变
   - **New**: 统一设置为新版本
   - **Old**: 统一设置为旧版本
6. 点击 **Save All** 保存

## 💡 使用场景

### 场景1: 标记旧版本模板
```
需求: 将一批旧版本的模板标记为Old
操作:
1. 在模板列表中筛选出旧模板
2. 勾选这些模板
3. 批量编辑 → Version Status 选择 "Old"
4. 保存
```

### 场景2: 查找所有新版本模板
```
需求: 查看所有正在使用的新版本模板
操作:
1. 访问搜索页面或模板列表
2. Version筛选选择 "New"
3. 查看结果
```

### 场景3: 版本迁移
```
需求: 将某些模板从旧版本升级到新版本
操作:
1. 筛选 Old 版本模板
2. 勾选需要升级的模板
3. 批量编辑 → Version Status 选择 "New"
4. 保存
```

### 场景4: 对比新旧版本
```
需求: 对比新旧版本模板的差异
操作:
1. 先筛选 New，查看并记录
2. 再筛选 Old，查看并对比
3. 或使用 All 查看所有，通过Version列区分
```

## 📊 视觉标识

### 版本状态徽章
- **New**: 绿色徽章 🟢
- **Old**: 橙色徽章 🟠

### 表格列位置
在搜索结果和模板列表中，**Version** 列位于：
- File Path
- Template Name  
- **Version** ← 新增
- Fields (仅模板列表)
- Project
- Status
- Description

## ⚠️ 注意事项

1. **版本状态与废弃状态的区别**:
   - **Version Status (New/Old)**: 表示模板的版本新旧
   - **Status (Active/Deprecated)**: 表示模板是否还在使用
   - 两者独立，可以组合使用，例如：
     - New + Active: 新的正在使用的模板
     - Old + Active: 旧的但仍在使用的模板
     - Old + Deprecated: 旧的已废弃的模板

2. **批量编辑时的选择性更新**:
   - 批量编辑时，如果Version Status选择"Keep original"，则不修改原有版本状态
   - 只有明确选择New或Old才会更新

3. **筛选与批量编辑结合**:
   - 可以先筛选出特定版本的模板
   - 然后全选进行批量编辑
   - 提高工作效率

## 🚀 快捷操作

### 快速查看所有新版本
```
1. 访问 /templates?version_status=new
2. 或直接访问后选择New筛选
```

### 快速将所有选中模板标记为旧版本
```
1. 勾选模板
2. 批量编辑
3. Version Status → Old
4. Save All
```

### 重置版本状态
```
如果批量编辑时误操作：
1. 重新勾选模板
2. 批量编辑
3. Version Status 选择正确的值
4. Save All
```

## 📈 统计信息

模板列表页面底部显示：
- **Total**: 当前筛选条件下的模板总数

这可以帮助你了解：
- 有多少个新版本模板
- 有多少个旧版本模板
- 总体模板数量

## 🔧 技术实现

### 数据库字段
```sql
ALTER TABLE json_files ADD COLUMN version_status TEXT DEFAULT 'new'
```

### API端点
- 搜索和查询支持 `version_status` 参数
- 单个更新: `/update_attributes` 支持 `version_status` 字段
- 批量更新: `/batch_update_attributes` 支持 `version_status` 字段

### 前端实现
- 筛选器: 下拉框选择，自动跳转或提交
- 徽章显示: 根据值显示不同颜色的标签
- 编辑表单: 包含版本状态选择字段
